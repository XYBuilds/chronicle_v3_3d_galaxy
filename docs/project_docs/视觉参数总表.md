## 11. Uniform 级条目统计与 dev-only GUI 建议

### 11.1 统计口径说明

- **口径 A — GLSL `uniform` 声明条数（按名称去重、按材质计）**  
  - 宏观 `ShaderMaterial`：**9**（`uPixelRatio`, `uSizeScale`, `uZCurrent`, `uZVisWindow`, `uFocusSizeMul`, `uBgSizeMul`, `uLMin`, `uLMax`, `uChroma`）— 见 `point.vert.glsl:1-9` 与 `galaxy.ts` uniforms 段（约 `62-71` 行）。  
  - 微观 `ShaderMaterial`：**14**（`uColor0`–`uColor3`, `uWeight0`–`uWeight3`, `uTime`, `uAlpha`, `uScale`, `uOctaves`, `uPersistence`, `uThreshold`）— 见 `perlin.frag.glsl:1-19` 与 `planet.ts:55-70`；顶点文件 `perlin.vert.glsl` 额外声明的 `uTime` 与材质字典为同一槽位，**不重复计**。  
  - **口径 A 合计：23 个 uniform 槽位。**

- **口径 B — P7.3「外观扫参」常调项（排除数据驱动色/权、排除自动同步的 `uPixelRatio`）**  
  宏观：`uSizeScale`, `uFocusSizeMul`, `uBgSizeMul`, `uLMin`, `uLMax`, `uChroma`, `uZVisWindow`（**7**；`uZCurrent` 更偏导航但仍影响构图，若纳入则 **8**）  
  + Bloom：`strength`, `radius`, `threshold`（**3**）  
  + 微观噪声：`uScale`, `uOctaves`, `uPersistence`, `uThreshold`（**4**）  
  + 常改 TS/片元隐式：点边缘 `smoothstep` 起止、`500` 透视因子、Perlin 时间系数与 rim（若纳入可再 **+5** 量级）  
  - **仅算 uniform 型旋钮（口径 B 窄）：7 + 3 + 4 = 14 项（若含 `uZCurrent` 为 15）——已 ≥ 10。**

### 11.2 是否建议引入 dev-only GUI（leva / lil-gui）

依据 Phase 7 计划约定：**uniform 级可调项 ≥ 10 → 建议引入 `import.meta.env.DEV` 守卫的 dev-only 面板。**

- 按 **口径 B（窄）**：**14（或 15）≥ 10 → 建议引入 dev-only GUI**，以免 P7.3 在源码与控制台间反复切换。  
- **缓解因素**：Storybook `GalaxyThreeLayerLab` 已提供一组 Controls，且生产挂载已有 `window.__bloom` / `__galaxyPointScale` / `__galaxyColor`。若团队 **固定只在 Storybook + 控制台** 扫参，可暂不引入 leva，但仍算「超过 10 个关注点」，文档层面仍记 **≥10**。  
- **结论（交付验收用语）**：**uniform 槽位（口径 A）= 23；P7.3 核心外观旋钮（口径 B）= 14（不含 `uZCurrent`）或 15（含）→ 满足「≥10」条件，按计划建议引入 dev-only GUI；若坚持零依赖，则以 Storybook Controls + 现有 `window.__*` 为替代方案，并在 P7.3 实施时显式写入选型记录。**  
- **实施记录（P7.3）**：已采用 **`leva`**，入口 `GalaxyThreeLayerLab.tsx`（`import.meta.env.DEV` + lazy），面板实现 `GalaxyThreeLayerLabLevaHost.tsx`。

### P8.0.2 双 InstancedMesh 压力 Story（P8.4 准入门槛）

- **入口**：Storybook → `Dev/Instanced mesh bench (P8.0)` → `DualMeshWorstCase`。源码：`frontend/src/storybook/InstancedMeshBench.tsx`。
- **内容**：同屏两个 `InstancedMesh`，各 **60 000** 实例；几何为 `IcosahedronGeometry(1, 0)` 与 `IcosahedronGeometry(1, 1)`；**相同** `instanceMatrix`（模拟最坏 overdraw）；材质为 `MeshBasicMaterial`（无 OKLab / 自定义片元），idle 侧 `transparent + !depthWrite` 近似未来 idle mesh。
- **Three.js r183 几何说明**：当前 `IcosahedronGeometry` 为**非索引**展开，`position.count` 为 **60**（detail 0）与 **240**（detail 1）每实例，故两 draw 合计约 **60 000×(60+240) = 18×10⁶** 次顶点属性参与 / 帧（高于拓扑意义上的 12+42 估算；若后续改为索引化几何，须重测并重写本段）。
- **理论备忘（拓扑顶点，仅供参考）**：`60k×12 + 60k×42 = 3.24M`（计划原文）— 与引擎实际 buffer 可能不一致，以本 Story **HUD + `renderer.info`** 为准。
